# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Static site for katiecrafts.com on Astro 6, content imported from a legacy WordPress site. Output is a fully static `dist/` deployed via GitHub Actions to GitHub Pages with a `CNAME`. Site canonical: `https://www.katiecrafts.com`. `trailingSlash: 'always'`.

## Commands

- `npm run dev` ‚Äî Astro dev server.
- `npm run build` ‚Äî static build into `dist/`.
- `npm run preview` ‚Äî serve `dist/`.
- `npm run deploy` ‚Äî `build` then copy `CNAME` into `dist/` (local preview of the deploy artifact).
- `npm run new-post -- --title "My Post"` ‚Äî scaffold a new post under `src/content/blog/<slug>/index.md`. Add `--slug <slug>` to override the auto-generated slug.
- `npm run lint` / `npm run lint:fix` ‚Äî oxlint (ignores in `.oxlintignore`).
- `npm run typecheck` ‚Äî `tsc --noEmit` against `astro/tsconfigs/strict`.

WordPress migration pipeline (one-shot tooling, retained for re-imports):
- `npm run tool:import-wordpress` ‚Äî fetches `/wp-json/wp/v2/posts?_embed` from `www.katiecrafts.com`, writes `data/posts/<slug>/{post.json,wordpress.json,images/}` and updates `data/import-manifest.json`. Flags: `--dry-run`, `--start-page=N`, `--max-pages=N`.
- `npm run tool:cleanup-posts -- --all` (or `--slug <slug>`) ‚Äî runs the post HTML through unified (rehype-parse ‚Üí cleanup passes ‚Üí rehype-remark ‚Üí remark-gfm ‚Üí remark-stringify) and writes the resulting Markdown back into `post.json`. Warns when the output still contains non-trivial raw HTML.
- `npm run tool:export-posts` ‚Äî emits `src/content/blog/<slug>/index.md` plus co-located images from `data/posts/`. Rewrites body image refs (markdown + raw `<img>`) to local filenames.

`data/posts/` and `data/import-manifest.json` are gitignored ‚Äî they are scratch state for the import pipeline. The published source of truth is `src/content/blog/`.

## Content model

Posts live in the `blog` content collection at `src/content/blog/<slug>/index.md`. The directory name is the slug. Schema is defined in `src/content.config.ts` and validates frontmatter:

```yaml
title: "..."
publishedDate: "YYYY-MM-DD"
excerpt: "..."          # optional
featuredImage:           # optional
  src: "./featured.jpg"  # resolved by image() helper
  alt: "..."
categories: [{slug, name}]
tags: [{slug, name}]
```

Images are co-located with `index.md` (no `_images/` subdir). The schema's `image()` helper turns `featuredImage.src` into an `ImageMetadata` object that `<Image />` from `astro:assets` consumes ‚Äî Sharp produces optimized variants at build time. Body images use relative `./<file>` paths and Astro's markdown processor handles them the same way.

## Routes

- `/` (`src/pages/index.astro`) ‚Äî hero + latest 3 + 6 category cards.
- `/blog/` and `/blog/page/[page]/` ‚Äî paginated archive (6 per page).
- `/blog/<slug>/` ‚Äî generated by `src/pages/blog/[...slug].astro` from `getCollection('blog')`. Computes prev/next at build time; renders byline, share buttons, and prev/next nav.
- `/category/`, `/category/<slug>/`, `/category/<slug>/page/<page>/` ‚Äî same triple under `/tag/`.
- `/rss.xml` ‚Äî latest 50 posts via `@astrojs/rss`.
- `/search-index.json` ‚Äî MiniSearch index served as JSON, fetched lazily by the header `Search` component on first input focus.
- `/legal/`, `/thanks/`, `/404`.

Taxonomy slugs come from each post's frontmatter `categories[]` / `tags[]` arrays. Adding a new term anywhere causes a new route to be generated.

## Utilities

- `src/utils/pagination.ts` ‚Äî `POSTS_PER_PAGE` (6), `HOME_POSTS_LIMIT` (3), `paginatePosts<T>()`, `formatPublishedDate()`.
- `src/utils/taxonomy.ts` ‚Äî `BlogEntry` alias for `CollectionEntry<'blog'>`, `sortByPublishedDate`, `getCategorySummaries`/`getTagSummaries`, `getPostsByCategory`/`getPostsByTag`.

There is intentionally no central `posts.ts`. Pages call `getCollection('blog')` directly and apply these helpers.

## Path aliases

Defined in both `tsconfig.json` (for TS) and `astro.config.mjs` `vite.resolve.alias` (for runtime). Keep them in sync:

`~` ‚Üí `src`, `~components`/`~layouts`/`~utils`/`~styles` ‚Üí `src/*`; `~data` ‚Üí `data`; `~tools` ‚Üí `tools`.

## Conventions

- Post bodies are Markdown (the WordPress HTML was converted in `tools/cleanup-posts.ts`). Authoring is plain Markdown.
- `Layout.astro` owns Open Graph / Twitter card / canonical / `article:*` meta and the `<link rel="alternate" type="application/rss+xml">`. Pass props rather than hand-writing `<head>` content in pages.
- `[...slug].astro` is the canonical post template; there is no separate `Post.astro` layout. Don't add `layout:` to post frontmatter.
- Excerpts may contain HTML (legacy WP excerpts) and are injected with `set:html` in `PostCard.astro` ‚Äî trusted because they originate from our own export.
- TypeScript is strict; oxlint runs over the whole tree minus `.astro/`.

## Deploy

Pushes to `main` trigger `.github/workflows/deploy.yml`, which builds the site and publishes to GitHub Pages via `actions/deploy-pages@v5` (OIDC, no `gh-pages` branch). The repo's Pages source must be set to "GitHub Actions" once in repo settings; `CNAME` is committed at the repo root and copied into `dist/` during the workflow.
